About This Project

SSL and TLS is used in securing our communications to many of the web sites we use -- Facebook, Google, your bank, your education institution. It is an open industry standard, but that does not prevent it from having occasional mistakes in their design. As with all other software, when such mistakes are found, update are issued by the manufacturer and the world moves on. However, with the increasingly diverse types and number of devices which we now possess, it is possible that our devices do not receive all the updates necessary to resolve these problems.

Repercussions

What does this mean for us? At the best case ‐ nothing at all. We hope that these standards are safe the first time it is designed. But very often it is not, thus there is a possibility that some of our devices have got security loopholes in the software.

One of the things which we use the most are our web browsers. These programs have an embedded SSL and TLS library as well. However, even with advances in the TLS specification, it has taken an average of (x?) years for browsers to implement them and reach the masses. This project tries to capture the various browsers that are in use today, and determine the proportion of them which implement the latest version of the TLS standard.

One must also remember that in any protocol there are two implementations involved: the user-facing side, which is the web browser, or your mobile app; as well as the back-end which the user does not directly interact with. As such, this project will attempt to analyse both the client and server ends of the SSL/TLS implementation.

For the server-side, I will be finding a list of sites that Singaporeans frequently access, and determine possible improvements in their site configuration. Alexa maintains a list of popular sites (ordered by hits) that I will be using as a reference to determine popularity. I will then run SSLabs' excellent security test suite on those sites identified, and compile them here.

Geographical Limits

At this point, I am restricting the study to only residents of Singapore, because the catalyst for this project is my coursework at the National University of Singapore. But there's going to be scope for a wider survey. SSLabs has some information that can be used as a starting point, and as mentioned, their excellent SSL Tool can help site administrators achieve a robust configuration for their secure web sites.

Participation

This project will use some techniques to try to identify the technologies both servers and client-side programs are using. They will be anonymous (see the Privacy Policy for details) and require almost no effort on your part. However, the aggregate data coming from all visitors can be used to draw conclusions which we may not currently be aware of, especially when formulating recommendations and best practices.
